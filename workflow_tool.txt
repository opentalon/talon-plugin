Execute a Talon workflow block — a deterministic multi-step MCP chain. Use this tool instead of calling tools one-by-one when you need to:

- Batch operations across many items (delete N items, assign N items to a person, update N tickets)
- Chain MCP calls whose later steps depend on earlier results
- Auto-paginate a "list" call until all items are collected

The runtime handles pagination, ID collection, step ordering, and per-step error handling. Write a Talon workflow block as a single string and pass it as the "workflow" argument.

Syntax:

```
workflow "short description" {
  step "step_name" {
    mcp "<server>" "<tool>" {
      <arg_name> <value>
      <arg_name> <expression>
    }
  }

  step "next_step" depends_on "step_name" {
    mcp "<server>" "<tool>" {
      <arg_name> step("step_name").result.<field>
    }
  }
}
```

Example — delete all items whose name contains "Test":

```
workflow "Delete all Test items" {
  step "find" {
    mcp "inventory" "list-items" {
      query "name:*Test*"
      collect_all true
    }
  }
  step "delete" depends_on "find" {
    mcp "inventory" "batch-delete" {
      ids step("find").result.items.map(id)
    }
  }
}
```

Example — onboard a new person and assign equipment:

```
workflow "Onboard new team member" {
  step "create_person" {
    mcp "hr" "create-person" {
      first_name context.first_name
      last_name context.last_name
    }
  }
  step "assign_equipment" depends_on "create_person" {
    mcp "inventory" "assign-item" {
      item_id context.laptop_id
      person_id step("create_person").result.id
    }
  }
}
```

Notes:

- `collect_all true` auto-paginates by re-calling the same MCP tool with `page` incremented until `has_more` is false
- `step("name").result.field` references a previous step's result
- `.map(id)` extracts a single field from a list (e.g. `result.items.map(id)` → `[1,2,3]`)
- `context.<name>` reads a value from the conversation/request context
- Step names must be unique within a workflow
- The workflow description should be 5-12 words
